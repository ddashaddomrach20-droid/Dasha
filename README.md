# Мессенджер

Простой мессенджер в стиле Telegram с реальной перепиской между пользователями.

## Возможности

- ✅ Регистрация по ФИО и паролю
- ✅ Поиск пользователей по имени
- ✅ Добавление в контакты
- ✅ Real-time переписка (WebSocket)
- ✅ История сообщений сохраняется в базе данных

## Запуск локально

### 1. Установка зависимостей сервера

```bash
cd server
npm install
```

### 2. Запуск сервера

```bash
npm start
```

Сервер запустится на `http://localhost:3000`

### 3. Открыть клиент

Просто откройте файл `index.html` в браузере (дважды кликните по файлу).

## Как пользоваться

1. **Зарегистрируйтесь** — введите ФИО и пароль (минимум 4 символа)
2. **Найдите контакт** — в поиске введите имя человека (например, "Алексей")
3. **Добавьте в контакты** — нажмите "Добавить"
4. **Начните переписку** — выберите чат и отправьте сообщение

## Деплой на хостинг

### Сервер (Render.com)

1. Зарегистрируйтесь на [render.com](https://render.com)
2. Создайте новый **Web Service**
3. Подключите ваш GitHub репозиторий
4. Настройки:
   - Root Directory: `server`
   - Build Command: `npm install`
   - Start Command: `npm start`
5. В Environment Variables добавьте (если нужно):
   - `PORT`: 3000

### Фронтенд (GitHub Pages)

1. В файле `script.js` замените `SERVER_URL` на URL вашего сервера:
   ```javascript
   const SERVER_URL = 'https://your-app.onrender.com';
   ```
2. Запушьте код на GitHub
3. Включите GitHub Pages в настройках репозитория

## Структура файлов

```
мессенджер/
├── index.html          # Главная страница
├── styles.css          # Стили
├── script.js           # Клиентский JavaScript
└── server/
    ├── server.js       # Сервер (Node.js + Express + Socket.io)
    ├── package.json    # Зависимости сервера
    └── messenger.db    # База данных (создаётся автоматически)
```

## Технологии

- **Фронтенд**: HTML, CSS, Vanilla JavaScript
- **Бэкенд**: Node.js, Express, Socket.io
- **База данных**: SQLite (better-sqlite3)
- **Аутентификация**: bcryptjs (хеширование паролей)
